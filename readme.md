# Описание

- `World` - "мир", где храниться абсолютно все: набор компанентов, контейнры с архетипами, сами сущности, системы.
- `World::new()` - инициализация мира.
- `World::spawn()` - выделяет новый `id` для сущности и ведёт учёт `generation`, чтобы при переиспользовании `id` не "воскрешать" прошлую сущьность. Метод возращает `Entity`.
- `World::despawn()` - проверяет, что сущность по `id` жива, удалет сущность из архетипа, увеличивает `generation` в `entities`. Возращает `bool` если удаление удачно - `true`, такой живой сущность не найдено - `faulse`.
- `World::insert_component()` - находит нужный архетип для сушности, если компанент не существует - перемещает в новый архетип с этим компонентом, кладёт компонент в нежный вектор в архетипе.
- `Archetype::add_component()` - кладёт в `HashMap` вектор копонентов под ключом `TypeId`.
- `Archetype::get_component_mut()` - извлекает вектор компонентов нужного типа из `HashMap`, приводит тип с помощью `downcast_mut`.

## Битовая адресация к наиболее частым архетипам:

Возможно держать ссылки на сущности внутри одно архетипа по битовому числу "тегов". 
Пример набора параметров:
1. движимый
2. имеет хп
3. имеет ИИ
Тогда: игровой НПС-моб будет лежать в 7 ячейке массива (так как 0b111 as usize = 7), дверь, что можно сломать лежит в 2 ячейке (так как 0b010 as usize = 2)
